Things you should know in Web Development

Learn Shell Basics

A skill you should pick up on day one is being able to use the shell. Shells are used as the command line utility of many operating systems such as Unix, Linux, and Mac. The most common shell type is called Bash. A good portion of web development tutorials on the web will deal with the command line in one way or another, therefore it’s a key skill to have.

Learn Git

Git is the most popular tool for controlling the versions of your projects. It’s what’s being used for this project and millions more on the web. Git is a tool you should know especially if you plan on working on a team that shares a codebase. Knowing some of the basic commands such as ‘clone’, ‘pull’ and ‘push’ can only get you so far, but they should be a good starting point if you want to dive in deeper.

Learn a backend

A backend language usually runs on the server and contains logic you don’t expose to the user. Some languages that are consider a backend are, Java, NodeJS, PHP, Go, Ruby, and Python. Two very popular languages being used today are NodeJS and Go. Backend languages are very versatile and can be implemented into any front end you pick, which in most cases will be Javascript, HTML and CSS.

Learn Javascript

Learning Javascript is vital to when it comes to webpages. Webpages have a something called a DOM which is short for Document Object Model. The DOM refers to how the page is loaded and laid out in code. Javascript is considered the language of the DOM and enables you to manipulate the DOM in any way you can. Things such as loading events, adding animations, filling in content, adding and removing content are just some examples of what you can do to the DOM. Harnessing the power of the webpage will let you have control over it, therefore allow you to do whatever you please.

Use Browser Tools

Learning how to use your browser’s tools will help tremendously in planning out a webpage and debugging it. You can directly manipulate objects as well as use the console to help you with errors that might arise. Firefox was the first browser to introduce a tool called Firebug to help web developers out. Today, all modern web browsers have browser tools. Investing your time to learning these tools will save you tons of time and stress!

Pick an IDE

An IDE is an Integrated Development Environment, which is simply a text editor for developers. They allow you to visually organize your code and display your files in a tree format for easy navigation. A popular IDE today that is open source is called Atom. Atom, made by GitHub allows for the use of plugins which developers have written to help wit things such as shortcuts and workflow.

Be familiar with jQuery

Javascript is great to have on your tool belt, although using a Javascript framework called jQuery is also good to be knowledgable in. jQuery allows you use the same functionality Javascript has, but be able to write less code and sometimes even do more with it. jQuery follows a simple to use and read API that anyone dipping their toes in web development can use right away.

Write tests

With all the code you will be writing, it’s important to be testing your code in parallel. It’s a good habit to write tests for your code so that you can better understand how it’s working and so that you can be almost certain that it will not contain any bugs and be an issue for other pieces within your project. There are many testing frameworks out for selection for you to use based off of your language and framework you’re working in.
